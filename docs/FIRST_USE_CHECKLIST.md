# First Use Checklist — ระบบยงเฮง มหาชัย รีไซเคิล

> คู่มือใช้งานระบบสำหรับพนักงาน — ทำตามขั้นตอนทีละข้อ
> หากมีปัญหา ให้หยุดและแจ้ง admin ทันที

---

## 1. ก่อนเริ่มวัน (Daily Start)

- [ ] เปิดเบราว์เซอร์ → ไปที่ **https://st-yongheng-recycle.vercel.app**
- [ ] ล็อกอินด้วย username + password ของตัวเอง
- [ ] ดู **แดชบอร์ด** — เช็คยอดสต็อกคงเหลือ + ยอดซื้อ/ขายเมื่อวาน
- [ ] ถ้าเห็นตัวเลขผิดปกติ → แจ้ง admin ก่อนเริ่มงาน

---

## 2. ตอนสร้างบิลซื้อ (Buy Bill)

- [ ] กดแท็บ **"รับซื้อ"**
- [ ] เลือกสินค้าจากรายการ (ค้นหาชื่อสินค้า)
- [ ] ใส่ **น้ำหนัก** (กก.) — ต้องมากกว่า 0
- [ ] ใส่ **ราคา/กก.** — ต้องไม่ติดลบ
- [ ] กด **"เพิ่มรายการ"**
- [ ] ตรวจทานยอดรวมในตะกร้าก่อนบันทึก
- [ ] ถ้าเป็นเครดิต → ติ๊ก "ซื้อเชื่อ"
- [ ] กด **"บันทึกใบรับซื้อ"**
- [ ] จดเลขบิล (BUY-2569-xxxxx) เก็บไว้

⚠️ **ห้าม**:
- ห้ามใส่น้ำหนักติดลบ
- ห้ามใส่ราคาติดลบ
- ห้ามกดบันทึก 2 ครั้ง (รอ toast "บันทึกสำเร็จ" ก่อน)

---

## 3. ตอนสร้างบิลขาย (Sell Bill)

- [ ] กดแท็บ **"ขาย"**
- [ ] เลือกสินค้า (เฉพาะที่มีสต็อก > 0)
- [ ] ใส่ **น้ำหนัก** — ต้องไม่เกินสต็อกคงเหลือ
- [ ] ใส่ **ราคาขาย/กก.** — ต้องมากกว่า 0
- [ ] กด **"เพิ่มรายการ"**
- [ ] ตรวจทานยอดขาย + กำไรในตะกร้า
- [ ] เลือกลูกค้า (ถ้ามี)
- [ ] กด **"บันทึกใบขาย"**
- [ ] จดเลขบิล (SELL-2569-xxxxx) เก็บไว้

⚠️ **ห้าม**:
- ห้ามขายเกินสต็อก — ระบบจะบล็อกเอง
- ห้ามตั้งราคาขาย = 0

---

## 4. ตอนแก้บิล (Edit Bill)

- [ ] ไปแท็บ **"ประวัติ"**
- [ ] ขยายบิลที่ต้องการแก้
- [ ] กดปุ่ม **"แก้ไข"** (ดินสอ)
- [ ] แก้ไขเฉพาะ field ที่อนุญาต:
  - บิลซื้อ: แก้ได้ครบ (วันที่, เครดิต, หมายเหตุ, items)
  - บิลขาย: แก้ได้แค่ **ราคาขาย/กก.**
  - บิลคัดแยก: แก้ได้แค่ **ราคารับซื้อที่คัดได้**
- [ ] กด **"บันทึก"**

⚠️ **ห้าม**:
- ห้ามแก้บิลเก่าเกิน 7 วันโดยไม่แจ้ง admin
- ห้ามแก้น้ำหนักในบิลขาย (ต้อง cancel + สร้างใหม่)

---

## 5. ตอนยกเลิกบิล (Cancel Bill) — ใช้แทน "ลบ"

- [ ] ไปแท็บ **"ประวัติ"**
- [ ] ขยายบิลที่ต้องการยกเลิก
- [ ] กดปุ่ม **"ลบ"** (ถังขยะ) — ระบบจะถามเหตุผล
- [ ] ใส่เหตุผลการยกเลิก (เช่น "กรอกผิด น้ำหนักผิด")
- [ ] กดยืนยัน

✅ **สิ่งที่เกิดขึ้น**:
- บิลถูกทำเครื่องหมาย "ยกเลิกแล้ว" (แดง)
- สต็อกถูกคืนอัตโนมัติ
- บิลยังอยู่ในประวัติ (audit trail)
- ยอดรวมใน dashboard ไม่นับบิลยกเลิก

⚠️ **ห้าม**:
- ห้ามยกเลิกบิลซื้อที่สินค้าถูกขายไปแล้ว (ระบบจะบล็อก)
- ห้ามยกเลิกบิลเพื่อ "แก้" — ให้ใช้ "แก้ไข" แทน

---

## 6. ตอนปิดวัน (Daily Close)

- [ ] ดู **แดชบอร์ด** — เช็คยอดซื้อ/ขายวันนี้
- [ ] ไปแท็บ **"สต็อก"** — เช็คน้ำหนักสต็อกรวม
- [ ] ไปแท็บ **"เครดิต"** — เช็คค้างรับ/ค้างจ่าย
- [ ] ถ้ามีบิลที่ยกเลิก → ตรวจว่าสต็อกคืนถูกต้อง
- [ ] แจ้ง admin ถ้ามีความผิดปกติ

---

## 🚨 สิ่งที่ต้องแจ้ง admin ทันที

1. ล็อกอินไม่ได้
2. สต็อกติดลบ
3. บิลซ้ำ (เลขบิลซ้ำ)
4. ยอดรวมใน dashboard ผิดปกติ
5. เห็นบิลที่ไม่ได้สร้างเอง
6. ลบบิลไม่ได้ / ยกเลิกไม่ได้
7. หน้าจอค้าง / error แดง
