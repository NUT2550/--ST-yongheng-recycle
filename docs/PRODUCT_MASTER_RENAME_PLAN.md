## 3-Phase Sync Strategy

### Phase 1: Rename existing products (40 รายการ)
- เปลี่ยนชื่อสินค้าเดิมให้ตรงระบบเก่า
- **preserve productId** — ไม่กระทบ stock/bill history
- ใช้ `UPDATE "Product" SET name = 'newName' WHERE id = 'productId'`
- ต้องระวัง `@unique` constraint — เช็คว่าชื่อใหม่ไม่ซ้ำกับสินค้าอื่น

### Phase 2: Create new products + categories (57 สินค้า + 2 หมวดใหม่)
- สร้าง category ใหม่: อิเล็กทรอนิกส์, พลาสติก
- สร้างสินค้าใหม่ 57 รายการ (ใช้ชื่อระบบเก่าโดยตรง)
- ใช้ `INSERT INTO "Product" ...` (additive only)

### Phase 3: Verify dropdowns
- ตรวจว่า Buy/Sell/Sort/Excel import dropdowns แสดงสินค้าครบ
- ตรวจว่า Sort source dropdown แสดงทุกหมวด (ไม่จำกัดเฉพาะเหล็ก)

## Risk Assessment

| Risk | Severity | Mitigation |
|------|----------|------------|
| Rename สินค้าที่มี stock ใหญ่ (เหล็กหล่อ 80: 70,000 กก.) | HIGH | preserve productId — stock ไม่กระทบ |
| Rename แล้วชื่อซ้ำกับสินค้าอื่น | MEDIUM | เช็คก่อน rename ด้วย findUnique |
| พนักงานสับสนกับชื่อใหม่ | LOW | ชื่อใหม่เป็นชื่อระบบเก่าที่คุ้นเคย |
| สินค้าซ้ำความหมาย (มุ้งลวด vs อลูมิเneียมมุ้งลวด) | LOW | owner ควร deactivate "มุ้งลวด" ภายหลัง |
