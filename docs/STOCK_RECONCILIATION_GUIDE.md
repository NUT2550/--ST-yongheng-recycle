# Stock Reconciliation Guide — คู่มือเทียบสต็อก

> ใช้สำหรับ admin เทียบสต็อกในระบบกับสต็อกจริงที่ลาน

---

## วัตถุประสงค์

ตรวจสอบว่าสต็อกในระบบตรงกับของจริงที่ลาน ถ้าไม่ตรง ต้องหาสาเหตุและแก้ไข

## ความถี่

- สัปดาห์แรก: ทุกวัน
- หลังจากนั้น: สัปดาห์ละ 1 ครั้ง (วันศุกร์หรือส่วนสุดท้ายของสัปดาห์)
- ก่อนปิดบัญชีรายเดือน: ทำ 1 ครั้ง

---

## ขั้นตอน

### 1. Export สต็อกจากระบบ

1. เปิดเว็บ → ล็อกอิน → ไปหน้า "สต็อก"
2. บันทึกยอดสต็อกรวมแต่ละสินค้า
3. หรือใช้ API: `GET /api/stock`

### 2. นับ/ชั่งของจริงที่ลาน

1. แยกตามประเภทสินค้า (เหล็ก/ทองแดง/อลูมิเนียม ฯลฯ)
2. ชั่งน้ำหนักแต่ละกอง
3. บันทึกในตารางเทียบ (ดู template ด้านล่าง)

### 3. เทียบส่วนต่าง

ใช้ตาราง:

| สินค้า | ระบบ (กก.) | จริง (กก.) | ส่วนต่าง | สาเหตุ |
|--------|-----------|-----------|---------|--------|
| หนาพิเศษ | 5,000 | 4,950 | -50 | ขายยังไม่ลงบิล |
| หนาสั้น | 54,914 | 55,000 | +86 | รับซื้อยังไม่ลงบิล |
| เหล็กคละ | 139 | 100 | -39 | คัดแยกยังไม่ลงบิล |

### 4. ระบุสาเหตุส่วนต่าง

สาเหตุที่เป็นไปได้:

| สาเหตุ | วิธีแก้ |
|--------|--------|
| รับซื้อยังไม่ลงบิล | ลงบิลรับซื้อในระบบ |
| ขายยังไม่ลงบิล | ลงบิลขายในระบบ |
| คัดแยกยังไม่ลงบิล | ลงบิลคัดแยกในระบบ |
| สูญเสีย/เน่าเสีย | ลงบิลคัดแยกแบบของเสีย |
| กรอกน้ำหนักผิด | ยกเลิกบิลเดิม + สร้างใหม่ |
| ลืมบิล | สร้างบิลเพิ่ม |
| ขโมย | แจ้ง admin + ตำรวจ |

### 5. สรุปผล

- ถ้าส่วนต่าง < 1% ของน้ำหนักรวม → ปกติ (ความคลาดเคลื่อนจากการชั่ง)
- ถ้าส่วนต่าง 1-5% → ต้องหาสาเหตุ + แก้ไข
- ถ้าส่วนต่าง > 5% → หยุดใช้ระบบ + ตรวจสอบทุกบิล
