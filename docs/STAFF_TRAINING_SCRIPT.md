# Staff Training Script — ระบบยงเฮง มหาชัย รีไซเคิล

> สคริปต์ฝึกพนักงาน — ใช้สำหรับ admin สอนพนักงานใหม่
> ระยะเวลาฝึก: ~30 นาที
> วัตถุประสงค์: พนักงานสามารถสร้างบิล + ตรวจประวัติ + ยกเลิกบิลได้ถูกต้อง

---

## ส่วนที่ 1: Login (5 นาที)

### สิ่งที่สอน:
1. เปิดเบราว์เซอร์ (แนะนำ Chrome หรือ Safari)
2. พิมพ์ URL: **https://st-yongheng-recycle.vercel.app**
3. ใส่ username + password ที่ admin ให้
4. กด "เข้าสู่ระบบ"

### ให้พนักงานทำ:
- [ ] ล็อกอินด้วยบัญชีของตัวเอง
- [ ] ลองล็อกอินผิด → ดู error message
- [ ] ลองกด "แสดงรหัสผ่าน" (ตา) เพื่อตรวจดู

### ข้อควรระวัง:
- ห้ามแชร์รหัสผ่าน
- ถ้าล็อกอินไม่ได้ → แจ้ง admin รีเซ็ตรหัส
- ออกจากระบบทุกครั้งเมื่อเลิกใช้

---

## ส่วนที่ 2: ดู Dashboard (5 นาที)

### สิ่งที่สอน:
1. หน้าแรกหลังล็อกอิน = **แดชบอร์ด**
2. ดูการ์ดสรุป:
   - สต็อกทั้งหมด (น้ำหนัก + มูลค่า)
   - สต็อกรับซื้อ vs คัดแยก
   - ยอดรับซื้อ/ขาย วันนี้
3. ดูตารางสินค้าแยกหมวดหมู่ (ขยายได้)
4. ดูบิลล่าสุด 5 รายการ

### ให้พนักงานทำ:
- [ ] บอกยอดสต็อกรวมตอนนี้
- [ ] บอกยอดซื้อวันนี้ (ถ้ามี)
- [ ] ขยายหมวด "เหล็ก" ดูสินค้าย่อย

### ข้อควรระวัง:
- ตัวเลขใน dashboard ไม่นับบิลที่ "ยกเลิกแล้ว"

---

## ส่วนที่ 3: สร้างบิลซื้อ (10 นาที)

### สิ่งที่สอน:
1. กดแท็บ **"รับซื้อ"**
2. เลือกสินค้า (ค้นหาในช่องค้นหา)
3. ใส่น้ำหนัก + ราคา/กก.
4. กด "เพิ่มรายการ"
5. ดูยอดรวมในตะกร้า
6. ตั้งวันที่ (default = วันนี้)
7. ถ้าเครดิต → ติ๊ก "ซื้อเชื่อ"
8. กด "บันทึกใบรับซื้อ"

### ให้พนักงานทำ (ใช้ข้อมูล test):
- [ ] สร้างบิลซื้อ "เหล็กหนาพิเศษ" 1 กก. @ 1 บาท
- [ ] ดูเลขบิลที่ได้ (BUY-2569-xxxxx)
- [ ] ไปดูสต็อก → น้ำหนักเพิ่ม 1 กก.

### ข้อควรระวัง:
- น้ำหนักต้อง > 0
- ราคาต้องไม่ติดลบ
- อย่ากดบันทึก 2 ครั้ง

---

## ส่วนที่ 4: ตรวจ History (5 นาที)

### สิ่งที่สอน:
1. กดแท็บ **"ประวัติ"**
2. เลือกประเภทบิล: รับซื้อ / ขาย / คัดแยก
3. ขยายบิลเพื่อดูรายละเอียด
4. ดูเลขบิล (BUY/SELL/SORT-2569-xxxxx)
5. ดูสถานะ:
   - บิลปกติ = ไม่มี badge
   - บิลเครดิต = badge "เครดิต" (เหลือง)
   - บิลยกเลิก = badge "ยกเลิกแล้ว" (แดง) + จาง + ขีดทับ

### ให้พนักงานทำ:
- [ ] หาบิลที่สร้างในส่วนที่ 3
- [ ] บอกเลขบิล + ยอดรวม
- [ ] ดูว่ามี badge "ยกเลิกแล้ว" ไหม

### ข้อควรระวัง:
- บิลยกเลิกยังอยู่ในประวัติ (audit trail) — ไม่ใช่บั๊ก
- บิลยกเลิกไม่นับในยอดรวม

---

## ส่วนที่ 5: ยกเลิกบิล (Cancel) — สำคัญมาก (5 นาที)

### สิ่งที่สอน:
1. **ห้ามใช้คำว่า "ลบ"** — ระบบใช้ "ยกเลิก"
2. ไปประวัติ → ขยายบิล → กดปุ่ม "ลบ" (ถังขยะ)
3. ใส่เหตุผลการยกเลิก
4. กดยืนยัน
5. ระบบจะ:
   - ทำเครื่องหมาย "ยกเลิกแล้ว"
   - คืนสต็อกอัตโนมัติ
   - เก็บบิลไว้ในประวัติ

### ให้พนักงานทำ:
- [ ] ยกเลิกบิล test ที่สร้างในส่วนที่ 3
- [ ] ใส่เหตุผล: "ทดสอบระบบ"
- [ ] ไปดูสต็อก → น้ำหนักลด 1 กก. (คืน)

### ข้อควรระวัง:
- ห้ามยกเลิกบิลจริงโดยไม่จำเป็น
- ถ้ากรอกผิด → ยกเลิก + สร้างใหม่ ดีกว่าแก้
- บิลยกเลิกไม่สามารถกู้คืนได้ (แต่สต็อกคืนแล้ว)

---

## ส่วนที่ 6: กฎเหล็ก (5 นาที)

### พนักงานต้องจำ:

1. **ห้ามแชร์รหัสผ่าน** — แต่ละคนมีบัญชีของตัวเอง
2. **ห้ามลบบิล** — ใช้ "ยกเลิก" เท่านั้น
3. **ห้ามแก้บิลเก่า 7 วัน** โดยไม่แจ้ง admin
4. **ถ้ากรอกผิด** → ยกเลิกบิล + สร้างใหม่
5. **ถ้าเน็ตหลุด** → รอเน็ตกลับ + refresh + ตรวจบิล
6. **ถ้าเห็นตัวเลขผิดปกติ** → หยุด + แจ้ง admin
7. **ถ้าไม่แน่ใจ** → ถาม admin ก่อนทำ

### สิ่งที่พนักงานทำได้:
- ✅ สร้างบิลซื้อ/ขาย/คัดแยก
- ✅ ดู dashboard, สต็อก, ประวัติ, เครดิต, โบนัส
- ✅ แก้บิล (ตามข้อจำกัด)
- ✅ ยกเลิกบิล (พร้อมเหตุผล)

### สิ่งที่พนักงานทำไม่ได้ (admin เท่านั้น):
- ❌ จัดการผู้ใช้ (เพิ่ม/ลบ/แก้)
- ❌ จัดการสินค้า (เพิ่ม/ลด/แก้)
- ❌ จัดการพนักงาน
- ❌ ลูกค้า (แก้/ลบ)

---

## ส่วนที่ 7: ทดสอบ (5 นาที)

### ให้พนักงานทำเอง (ไม่มี script):
1. ล็อกอิน
2. สร้างบิลซื้อ (สินค้าอะไรก็ได้, 1 กก., 1 บาท)
3. ไปดูสต็อก — ยอดเพิ่ม 1 กก.
4. ไปดูประวัติ — เจอบิลใหม่
5. ยกเลิกบิล (เหตุผล: "ทดสอบ")
6. ไปดูสต็อก — ยอดกลับเดิม
7. ไปดูประวัติ — บิลมี badge "ยกเลิกแล้ว"

### ผ่านเกณฑ์:
- [ ] ทำครบทุกขั้นตอนโดยไม่มี error
- [ ] สต็อกกลับเดิมหลังยกเลิก
- [ ] บิลยกเลิกแสดง badge แดง

---

## หลังฝึกเสร็จ

- ให้พนักงานอ่าน `FIRST_USE_CHECKLIST.md`
- บอกพนักงานว่าระบบยัง "ทดสอบ" — ถ้ามีปัญหาให้แจ้ง admin ทันที
- 3 วันแรก: admin ควรเช็คสต็อก + audit log ทุกวัน
