# Controlled Rollout Checklist — 7 วันแรก

> ใช้สำหรับ admin (01/นัท) ตรวจสอบระบบทุกวันในสัปดาห์แรกของการใช้งานจริง

---

## ทุกเช้าก่อนเริ่มงาน

- [ ] เปิดเว็บ https://st-yongheng-recycle.vercel.app
- [ ] ล็อกอินด้วยบัญชี 01
- [ ] ดูแดชบอร์ด — เช็คยอดสต็อกรวม + ยอดซื้อ/ขายเมื่อวาน
- [ ] ถ้าเห็นตัวเลขผิดปกติ → หยุด แจ้งทีมซ่อม

## ทุกเย็นก่อนเลิกงาน

- [ ] ดูแดชบอร์ด — เช็คยอดซื้อ/ขายวันนี้
- [ ] ไปหน้าสต็อก — เช็คน้ำหนักสต็อกรวม
- [ ] ไปหน้าประวัติ — เช็คบิลที่สร้างวันนี้
- [ ] ถ้ามีบิลที่ cancel — ตรวจว่าสต็อกคืนถูกต้อง
- [ ] ถ้ามี error หรือตัวเลขผิด → บันทึก + แจ้งทีมซ่อม

## สิ่งที่ต้องเช็กทุก 2 ชั่วโมง (3 วันแรก)

- [ ] สต็อกไม่ติดลบ
- [ ] ไม่มีบิลซ้ำ (เลขบิลซ้ำ)
- [ ] ยอดรวมในแดชบอร์ดดูสมเหตุสมผล

## วิธีจัดการถ้าบิลผิด

### กรณี 1: กรอกน้ำหนัก/ราคาผิด
1. ไปหน้าประวัติ
2. ขยายบิลที่ผิด
3. กด "แก้ไข" (ดินสอ)
4. แก้ราคา/วันที่/หมายเหตุ
5. บันทึก

### กรณี 2: สินค้าผิด หรือน้ำหนักผิดมาก
1. ไปหน้าประวัติ
2. ขยายบิลที่ผิด
3. กด "ลบ" (ถังขยะ) — ระบบจะถามเหตุผล
4. ใส่เหตุผล (เช่น "กรอกผิด สินค้าผิด")
5. ยืนยัน — สต็อกจะคืนอัตโนมัติ
6. สร้างบิลใหม่ที่ถูกต้อง

### กรณี 3: ระบบ error / หน้าจอค้าง
1. หยุดใช้งาน
2. แจ้งทีมซ่อมทันที
3. อย่าลบ/แก้ข้อมูลจนกว่าจะได้รับคำแนะนำ

## สิ่งที่ต้องรายงานทันที

- สต็อกติดลบ
- บิลซ้ำ (เลขบิลซ้ำ)
- ยอดรวมในแดชบอร์ดผิดปกติ
- หน้าจอค้าง / error แดง
- ล็อกอินไม่ได้
- เห็นบิลที่ไม่ได้สร้างเอง

## วันที่ 7 — ประเมินรายสัปดาห์

- [ ] นับบิลทั้งหมดที่สร้างสัปดาห์นี้
- [ ] นับบิลที่ cancel — สาเหตุอะไร?
- [ ] เทียบสต็อกในระบบกับสต็อกจริงที่ลาน
- [ ] สรุปปัญหาที่เจอ → ส่งให้ทีมซ่อม
- [ ] ตัดสินใจ: ใช้จริงเต็มรูปแบบ หรือ ยังต้องปรับ?
