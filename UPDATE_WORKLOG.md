# วิธีอัปเดต Worklog (สำหรับ AI ทุกตัว)

> **ทุกครั้งที่ทำงานกับโปรเจกท์นี้ AI ต้องอัปเดต `/home/z/my-project/worklog.md`**

## กฎเหล็ก

1. **อ่าน `worklog.md` ก่อนเริ่มงานทุกครั้ง** — เพื่อเข้าใจสิ่งที่ทำไปแล้ว
2. **หลังเสร็จงาน ต้อง append บันทึกใหม่** ต่อท้ายส่วน "8. ประวัติการทำงาน"
3. **อัปเดตส่วนอื่นถ้ามีการเปลี่ยนแปลง** — เช่น เพิ่มฟีเจอร์ใหม่ → อัปเดตส่วน 2
4. **ห้ามลบไฟล์นี้** — เป็นเอกสาร disaster recovery

## รูปแบบบันทึกใหม่ (append ต่อท้ายส่วน 8)

```markdown
---
Task ID: <เลขถัดไป>
Agent: <ชื่อ agent>
Date: YYYY-MM-DD
Task: <สรุปงานที่ทำ>

Work Log:
- <ขั้นตอนที่ทำ 1>
- <ขั้นตอนที่ทำ 2>
- ...

Stage Summary:
- <ผลลัพธ์สำคัญ>
```

## ตัวอย่าง

```markdown
---
Task ID: 13
Agent: Main
Date: 2026-06-24
Task: เพิ่มฟีเจอร์ export Excel

Work Log:
- ติดตั้ง exceljs สำหรับ export

- อัปเดตส่วน 2.9 (history) ใน worklog

Stage Summary:
- ผู้ใช้สามารถ export บิลเป็นไฟล์ Excel ได้
```

## ส่วนที่ต้องอัปเดตตามสถานการณ์

| เหตุการณ์ | ส่วนที่อัปเดต |
|----------|--------------|
| เพิ่ม/แก้ฟีเจอร์ | ส่วน 2 (ฟีเจอร์) |
| เพิ่ม/แก้ API route | ส่วน 4 (API Routes) |
| เพิ่ม/แก้ Prisma model | ส่วน 3 (Database Schema) |
| เพิ่ม/แก้ไฟล์ | ส่วน 5 (โครงสร้างไฟล์) |
| เปลี่ยน config | ส่วน 6 (Configuration) |
| เปลี่ยน tech stack | ส่วน 1 (ภาพรวม) + ส่วน 7 (วิธีสร้าง) |
| พบเรื่องสำคัญ | ส่วน 9 (หมายเหตุ) |
| ทำงานเสร็จ | ส่วน 8 (ประวัติ) — เสมอ |
